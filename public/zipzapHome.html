Frequently Asked Question
Got a question? We've got answers.Can't find what you're looking for? Contact us!
Q1. What is ZipZap?
  • ZipZap is a self-service tool for migrating data from Gong to Zoom Revenue Accelerator (ZRA). It simplifies the migration process, allowing users to sign up and configure the tool to initiate data migration seamlessly.
  • To introduce the platform, ZipZap offers a free trial that includes migrating up to 100 recording files and a preview of the migrated data within ZRA. Once satisfied with the initial results, users can upgrade to the premium plan to complete a full migration of all historical data.
Q2.What data can ZipZap migrate?
  • Currently, ZipZap self-service migration tool is capable of transferring all your recording files from Gong to Zoom Revenue Accelerator (ZRA), along with their associated metadata and transcripts.
  • This ensures that your core call/meeting data is seamlessly migrated to the new platform. However, due to ZRA API limitations, some features such as recreating Gong folders as ZRA playlists, setting up Gong trackers as ZRA indicators, and establishing team hierarchies in ZRA is not supported by ZipZap at the moment and they have to be set up manually within ZRA after the migration.
Q3.Can ZipZap replicate my Gong company library folders in ZRA?
While ZRA doesn't directly replicate Gong's folder structure, it offers alternative methods to organize your recordings:
  • Basic playlists: Create playlists similar to Gong folders and manually add relevant call recordings by searching with call topic names.
  • Smart playlists: Create playlists with rules/filters to automatically add conversations based on criteria such as selected reps, selected customers, specified stages at the time
Q4.I have folders in Gong with Playlists, what is the process to get these playlists into ZRA? Do I need to create a folder first?
Currently, ZipZap does not support creating playlists and folders in ZRA due to API limitations.

However, a ZRA-licensed user in your organization can create playlists in ZRA to replicate your Gong playlists. These playlists can then be organized into folders for easier browsing.
  • Once ZipZap completes the file migration, you can create basic playlists and add relevant files by searching via conversation topic names.
  • Otherwise you can create smart playlists before migration so any migrated files matching the search criteria will be automatically added to the playlist.
Reference for more details: - ZRA Playlists
Q5.For the recorded conversations, do I need to set up the Team Hierarchy first, so then recorded conversations will have the required level of permission?
  • Any user with a ZRA license in your organization will have default view access permission to all recordings in ZRA.
  • However, your ZRA Admin can customize access permissions using ZRA's role management feature. This allows you to either select a default role or create a custom role with specific permissions tailored to your organization's needs. These roles can then be assigned to individual users.
Reference for more details- ZRA Role Management
Q6.Where does ZipZap store data during migration, and is it safe?
ZipZap's data handling process during migration is designed with security and privacy in mind:
  • During migration, Gong data is stored temporarily in the cache memory within the ZipZap application.
  • After successful migration to Zoom Revenue Accelerator (ZRA), all temporarily stored data is immediately cleared from ZipZap's cache.
  • ZipZap does not retain any customer recording files, associated metadata, or transcripts after the migration process is complete.
  • This approach ensures that the migration process is fully safe and secure and customer privacy is maintained throughout the entire process
This data handling protocol is designed to provide customers with confidence in the security and privacy of their information during the migration process from Gong to ZRA.
Q7.How many years' worth of data can I migrate using ZipZap?
  • ZipZap can migrate all your existing data from Gong. However, please be aware that Gong's default retention policy is 3 years. This means ZipZap cannot migrate data that has been automatically deleted due to exceeding this limit. If you require migration of older data, please contact Gong customer support to adjust your Gong retention policy.
Q8.What should I do if I cease being a Gong customer?
  • If you terminate your Gong subscription, your data will be irreversibly deleted within thirty (30) days. Ensure you back up your data to a secure storage solution before termination.
Reference for more details: - Gong Data Retention Policy
Q9.Does ZipZap support extracting data from cloud storage if my contract with Gong has expired and I have moved my data to cloud storage?
  • No, currently ZipZap does not support reading Gong data directly from cloud storage. This limitation exists for the following reasons:
    • Varied Cloud Platforms: Cloud storage accessibility and configurations differ significantly between vendors such as AWS, GCP, Azure, and others.
    • Standardization Challenges: The lack of a standardized interface across these platforms makes it difficult to implement a one-size-fits-all solution within ZipZap.
  • While ZipZap doesn't offer this feature, you can contact the Codimite Professional Services Team for assistance. They can assist you in creating a tailored solution to connect your cloud storage, extract Gong data, and successfully upload it to ZipZap.
Q10.Do I need to wait for the entire migration to finish before using the Zoom Revenue Accelerator?
No, you don't need to wait for the migration to complete before using ZRA. The process is designed to be flexible and user-friendly:
  • You can decide on a cut-off date in Gong. After this date, you can stop using Gong and initiate the migration process.
  • You can start using ZRA right away, without waiting for the migration to complete.
  • ZipZap will backfill your historical data in the background.
  • The migration process works chronologically, transferring recordings from your oldest to most recent.
This method allows for a seamless transition, minimizing disruption to your workflow while ensuring all your historical data is transferred to ZRA over time.
Q11.What should I do before initiating the migration process with ZipZap?
  • Before initiating the migration process, ensure to discontinue all activity on the Gong platform. Any new recordings added to Gong after the migration has started will not be transferred to ZRA.
Q12.How should I prepare my ZRA user accounts before migration?
  • Ensure all your Active Gong user accounts are migrated to Zoom and assigned the Zoom Revenue Accelerator (ZRA) licenses for them.
Q13.How do I sign up for ZipZap?
  • Users can sign up using the Google SSO (Single Sign-On) option or by using an email and password. Once signed up, users need to provide specific details to access Gong and ZRA, which are mandatory to execute the migration.
Q14.What details do I need to provide from the Gong side?
  • Developer API Key
  • Secret Key
  • API Base URL
Please note that you must have Technical Administrator privileges in Gong to obtain these credentials.

If you're unsure how to get this information, don't worry. When you reach this step in the ZipZap, you can refer to our step-by-step guide. The guide provides detailed instructions on how to obtain the necessary credentials.
Q15.Why should I provide Gong credentials to ZipZap?
  • To access your Gong call recordings, ZipZap utilizes Gong's APIs. In order to extract data from these APIs, ZipZap needs proper authorization from your Gong account to authenticate API calls. By providing your Gong credentials, you grant ZipZap the necessary permissions to securely access and retrieve your call recording data through Gong's API infrastructure.
  • ZipZap handles your credentials with the utmost security and uses them solely for the purpose of accessing your Gong data as part of its service offering.
Q16.What details do I need to provide from the Zoom side?
  • Account ID
  • Client ID
  • Client Secret
You must have Zoom Administrator privileges with a ZRA Admin Role to obtain these credentials.

If you're unsure how to get this information, don't worry. When you reach this step in the ZipZap setup process, you will have access to our step-by-step guide which provides detailed instructions on how to obtain the necessary credentials from your Zoom account.
Q17.Why should I provide Zoom credentials to ZipZap?
ZipZap utilizes Zoom Revenue Accelerator APIs to migrate your Gong call recordings to ZRA.
  • To upload data extracted from Gong via these ZRA APIs, ZipZap requires proper authorization from your Zoom account. This authorization allows ZipZap to authenticate its API calls to ZRA.
  • Your Zoom credentials enable ZipZap to establish a secure connection with your ZRA account.
  • Providing these credentials allows for a seamless, automated migration process without manual intervention for each data transfer.
Rest assured that ZipZap handles your Zoom credentials with the utmost security, using them solely for the purpose of facilitating the data migration from Gong to ZRA.
Q18.What steps do I need to follow after providing Gong and Zoom credentials to ZipZap?
  • After providing the required credential details from Gong and Zoom, you can analyze the data on the Gong side, such as the number of recordings and estimated years of data available in your account. Once the analysis is complete, you can start the migration.
Q19.What happens during the import process if a user no longer exists within Zoom or Gong?
  • Even if a user no longer exists on Gong or Zoom, ZipZap can still migrate your Gong recordings to the Zoom Revenue Accelerator (ZRA) using your Zoom admin account. After migration, these recordings will appear as it is owned by the Zoom admin account.
Q20.What happens if the meeting host doesn't have a ZRA license before migration?
  • If the meeting host doesn't have a ZRA license before migration, the meeting host will assign to your Zoom admin account by default. ZRA will still display the original host (the unlicensed ZRA user) as an internal participant in those meetings.
  • To maintain the original host information on the Zoom side, each meeting host needs to have an active ZRA license.
Q21.What happens if the meeting host is deactivated before migration (terminated gong user)?
  • If the meeting host is a deactivated user in Gong, ZipZap can still migrate their Gong recordings to ZRA using your Zoom admin account. After migration, these recordings will appear as they are owned by the Zoom admin account as well as Host name. However, ZRA will still display the original host (the deactivated user) as an internal participant in those meetings
Q22.Can I select a custom time period for migration?
  • No, ZipZap does not support custom time period selection before migrating data. Migration starts from your oldest Gong recording files.
Q23.Is the payment process safe when upgrading to the ZipZap premium plan?
  • Yes, the payment process is safe. ZipZap uses Stripe as the payment gateway, ensuring secure and reliable transactions. No card details are kept on our side.
